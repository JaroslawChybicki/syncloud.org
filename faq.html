Frequently Asked Questions

1) How can I access my Syncloud device?
2) Can I access my device from anywhere through Internet?
3) Is it possible to use HTTPS with Syncloud device?
4) Where Syncloud stores data?
5) What is the difference between redirect user and user?
How can I access my Syncloud device?
Each Syncloud device has a domain name associated with it. You provide this domain name during device activation. For example if you provided "mydomain" then your device should be accessible at http://mydomain.syncloud.it address. You can always get device address after logging into syncloud.it.
Can I access my device from anywhere through Internet?
Yes, you can. By default Syncloud device is in internal mode. That means it is not available outside of the home network. You can switch Syncloud device to external mode. This will make it available from anywhere through Internet as long as Syncloud device is connected to Internet.
Is it possible to use HTTPS with Syncloud device?
HTTPS can be enabled in the device settings panel. Right now only self-signed certificates are supported. This means that all browsers will alert you once you will try to access your device. Just ignore these alerts and keep using your device. We are working to improve this experience in future releases.
Where Syncloud stores data?
Syncloud stores all data either on the device itself or on attached external hard drive. This includes all data of Syncloud applications. It's not necessary to get external hard disk to start using Syncloud. There are always few free gigabytes on SD card or flash memory, this space is used by Syncloud as a storage.
What is the difference between redirect user and user?
Syncloud provides users with device domain name (DNS) which has to be maintained centrally. We use our own open source Dynamic DNS service "Redirect" which is hosting syncloud.it and redirect user/password is used to create domains and later in device activation procedure to connect devices with these domains. On the other hand we need another account for users to access their devices which they should not share with Syncloud Redirect service in any way. This device account is simply referred in activation process as user/password.
